NAME
fence_rcd_serial - rcd_serial fence agent

DESCRIPTION
fence_rcd_serial operates a serial cable that toggles a reset of an opposing server using the reset switch on its motherboard. The cable itself is simple with no power, network
or moving parts. An example of the cable is available here: https://smcleod.net/rcd-stonith/ and the circuit design is available in the fence-agents src as SVG

fence_rcd_serial accepts options on the command line as well as from stdin. Fenced sends parameters through stdin when it execs the agent. fence_rcd_serial can be run by itself
with command line options. This is useful for testing and for turning outlets on or off from scripts.

Vendor URL: http://www.scl.co.uk/rcd_serial/

PARAMETERS

-o, --action=[action]
Fencing action (Default Value: reboot)

-m, --method=[method]
Method to fence (onoff|cycle) (Default Value: cycle)

--serial-port=[port]
Port of the serial device (Default Value: /dev/ttyS0)

-q, --quiet
Disable logging to stderr. Does not affect --verbose or --debug logging to syslog.

-v, --verbose
Verbose mode

-D, --debug-file=[debugfile]
Write debug information to given file

-V, --version
Display version information and exit

-h, --help
Display help and exit

--delay=[seconds]
Wait X seconds before fencing is started (Default Value: 0)

--login-timeout=[seconds]
Wait X seconds for cmd prompt after login (Default Value: 5)

--power-timeout=[seconds]
Test X seconds for status change after ON/OFF (Default Value: 20)

--power-wait=[seconds]
Wait X seconds after issuing ON/OFF (Default Value: 2)

--shell-timeout=[seconds]
Wait X seconds for cmd prompt after issuing command (Default Value: 3)

--retry-on=[attempts]
Count of attempts to retry power on (Default Value: 1)

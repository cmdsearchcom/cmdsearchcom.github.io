NAME

gm-backup - complete backup of GNUmed database
 

SYNOPSIS

gm-backup
 

DESCRIPTION

The gm-backup script creates a backup of the database schema, data, and roles which can be used to recreate the backed up GNUmed database from scratch or to restore data into an existing GNUmed database (that is devoid of any data).
 
It is a really good idea to use it in a daily cron/anacron job to ensure that GNUmed patient data are safe against data loss. The Debian GNU/Linux packaged version of the GNUmed server installs such a cron job by default.

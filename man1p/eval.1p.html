NAME
eval - construct command by concatenating arguments

SYNOPSIS
eval [argument ...]

DESCRIPTION
The eval utility shall construct a command by concatenating arguments together, separating each with a <space>. The constructed command shall be read and executed by the shell.

OPTIONS
None.

OPERANDS
See the DESCRIPTION.

STDIN
Not used.

INPUT FILES
None.

ENVIRONMENT VARIABLES
None.

ASYNCHRONOUS EVENTS
Default.

STDOUT
Not used.

STDERR
The standard error shall be used only for diagnostic messages.

OUTPUT FILES
None.

EXTENDED DESCRIPTION
None.

EXIT STATUS
If there are no arguments, or only null arguments, eval shall return a zero exit status; otherwise, it shall return the exit status of the command defined by the string of conâ
catenated arguments separated by <space>s.

CONSEQUENCES OF ERRORS
Default.

The following sections are informative.

APPLICATION USAGE
None.

EXAMPLES
foo=10 x=foo
y=’$’$x
echo $y
$foo
eval y=’$’$x
echo $y
10

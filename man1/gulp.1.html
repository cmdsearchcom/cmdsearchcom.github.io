Flags
gulp has very few flags to know about. All other flags are for tasks to use if needed.

Â· -v or --version will display the global and local gulp versions

Â· --require <module path> will require a module before running the gulpfile. This is useful for transpilers but also has other applications. You can use multiple --require flags

Â· --gulpfile <gulpfile path> will manually set path of gulpfile. Useful if you have multiple gulpfiles. This will set the CWD to the gulpfile directory as well

Â· --cwd <dir path> will manually set the CWD. The search for the gulpfile, as well as the relativity of all requires will be from here

Â· -T or --tasks will display the task dependency tree for the loaded gulpfile

Â· --tasks-simple will display a plaintext list of tasks for the loaded gulpfile

Â· --color will force gulp and gulp plugins to display colors even when no color support is detected

Â· --no-color will force gulp and gulp plugins to not display colors even when color support is detected

Â· --silent will disable all gulp logging

The CLI adds process.env.INIT_CWD which is the original cwd it was launched from.

Tasks
Tasks can be executed by running gulp <task> <othertask>. Just running gulp will execute the task you registered called default. If there is no default task gulp will error.

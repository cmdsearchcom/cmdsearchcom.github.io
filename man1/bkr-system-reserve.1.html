NAME
bkr-system-reserve - Manually reserve a Beaker system

SYNOPSIS
bkr system-reserve [options] <fqdn>

DESCRIPTION
Reserves a Beaker system.

The system must have its status set to Manual and be free for use by the current user. To reserve a system using the Beaker scheduler, submit a job using the Beaker web UI or a
workflow command (such as bkr workflow-simple) instead.

OPTIONS
Common bkr options are described in the Options section of bkr(1).

EXIT STATUS
Non-zero on error, otherwise zero.

EXAMPLES
Reserve a particular system, provision it, do some work on it, and then release it:

bkr system-reserve system1.example.invalid
bkr system-provision --kernel-opts "nogpt" --distro-tree 12345 system1.example.invalid
# do some work on the system
bkr system-release system1.example.invalid

NAME

dynalogind - dynalogin server daemon

SYNOPSIS

dynalogind [ OPTIONS]...

DESCRIPTION

The dynalogin server daemon (dynalogin) listens on a TCP port (default localhost:9050) and processes HOTP validation requests. When a user is validated successfully, it updates the counter that must be retained as part of the HOTP algorithm. Credentials and algorithm data (such as the counter value) are kept in persistent storage using one of the modular storage backends (for example, flat file or UNIXODBC).

NAME

rootn.exe - ROOT interpretor memory managed backend

DESCRIPTION

This executable is linked with libNew, unlike root.exe, which makes it appropriate for if one wants to use memory mapped file, TMapFile, and so on. In all other aspects, it's the same as the root(1) executable.

SEE ALSO

root(1), root.exe(1), and cint(1)

NAME
fish -- the friendly interactive shell

Synopsis
fish [OPTIONS] [-c command] [FILE [ARGUMENTS...]]

Description
fish is a command-line shell written mainly with interactive use in mind. The full manual is available in HTML by using the help command from inside fish.

The following options are available:

Â· -c or --command=COMMANDS evaluate the specified commands instead of reading from the commandline

Â· -d or --debug-level=DEBUG_LEVEL specify the verbosity level of fish. A higher number means higher verbosity. The default level is 1.

Â· -i or --interactive specify that fish is to run in interactive mode

Â· -l or --login specify that fish is to run as a login shell

Â· -n or --no-execute do not execute any commands, only perform syntax checking

Â· -p or --profile=PROFILE_FILE when fish exits, output timing information on all executed commands to the specified file

Â· -v or --version display version and exit

Â· -D or --debug-stack-frames=DEBUG_LEVEL specify how many stack frames to display when debug messages are written. The default is zero. A value of 3 or 4 is usually sufficient to
gain insight into how a given debug call was reached but you can specify a value up to 128.

The fish exit status is generally the exit status of the last foreground command. If fish is exiting because of a parse error, the exit status is 127.

NAME
mojomojo_create.pl - Create a new Catalyst Component

SYNOPSIS
mojomojo_create.pl [options] model|view|controller name [helper] [options]

Options:
--force don’t create a .new file where a file to be created exists
--mechanize use Test::WWW::Mechanize::Catalyst for tests if available
--help display this help and exits

Examples:
mojomojo_create.pl controller My::Controller
mojomojo_create.pl -mechanize controller My::Controller
mojomojo_create.pl view My::View
mojomojo_create.pl view HTML TT
mojomojo_create.pl model My::Model
mojomojo_create.pl model SomeDB DBIC::Schema MyApp::Schema create=dynamic dbi:SQLite:/tmp/my.db
mojomojo_create.pl model AnotherDB DBIC::Schema MyApp::Schema create=static [Loader opts like db_schema, naming] dbi:Pg:dbname=foo root 4321
[connect_info opts like quote_char, name_sep]

See also:
perldoc Catalyst::Manual
perldoc Catalyst::Manual::Intro
perldoc Catalyst::Helper::Model::DBIC::Schema
perldoc Catalyst::Model::DBIC::Schema
perldoc Catalyst::View::TT

DESCRIPTION
Create a new Catalyst Component.

Existing component files are not overwritten. If any of the component files to be created already exist the file will be written with a ’.new’ suffix. This behavior can be
suppressed with the "-force" option.

NAME
strip â remove unnecessary information from strippable files (DEVELOPMENT)

SYNOPSIS
strip file...

DESCRIPTION
A strippable file is defined as a relocatable, object, or executable file. On XSI-conformant systems, a strippable file can also be an archive of object or relocatable files.

The strip utility shall remove from strippable files named by the file operands any information the implementor deems unnecessary for execution of those files. The nature of that
information is unspecified. The effect of strip on object and executable files shall be similar to the use of the âs option to c99 or fort77. The effect of strip on an archive
of object files shall be similar to the use of the âs option to c99 or fort77 for each object file in the archive.

OPTIONS
None.

file A pathname referring to a strippable file.

STDIN
Not used.

The input files shall be in the form of strippable files successfully produced by any compiler defined by this volume of POSIX.1â2008 or produced by creating or updating an arâ
chive of such files using the ar utility.

RATIONALE
Historically, this utility has been used to remove the symbol table from a strippable file. It was included since it is known that the amount of symbolic information can amount
to several megabytes; the ability to remove it in a portable manner was deemed important, especially for smaller systems.

The behavior of strip on object and executable files is said to be the same as the âs option to a compiler. While the end result is essentially the same, it is not required to be
identical.

XSI-conformant systems support use of strip on archive files containing object files or relocatable files.

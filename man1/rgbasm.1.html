NAME
rgbasm â Game Boy assembler

SYNOPSIS
rgbasm [-Ehv] [-b chars] [-D name[=value]] [-g chars] [-i path] [-o outfile] [-p pad_value] file

DESCRIPTION
The rgbasm program creates an object file from an assembly source file. Its arguments are as follows:

-b chars
Change the two characters used for binary constants. The defaults are 01.

-D name[=value]
Add string symbol to the compiled source code. This is equivalent to name EQUS "value" in code. If a value is not specified, a value of 1 is given.

-E Export all labels, including unreferenced and local labels.

-g chars
Change the four characters used for binary constants. The defaults are 0123.

-h By default, rgbasm inserts a ânopâ instruction immediately after any âhaltâ instruction. The -h option disables this behavior.

-i path
Add an include path.

-o outfile
Write an object file to the given filename.

-p pad_value
When padding an image, pad with this value. The default is 0x00.

-v Be verbose.

EXAMPLES
Assembling a basic source file is simple:

$ rgbasm -o bar.o foo.asm

The resulting object file is not yet a usable ROM image â it must first be run through rgblink(1) and rgbfix(1).

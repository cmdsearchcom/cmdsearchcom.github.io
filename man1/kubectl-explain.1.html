NAME

kubectl explain - Documentation of resources
 
 

SYNOPSIS

kubectl explain [OPTIONS]
 
 

DESCRIPTION

Documentation of resources.
 
Valid resource types include:
0[step?]
 
item buildconfigs (aka 'bc')
 
item builds
 
item clusters (valid only for federation apiservers)
 
item componentstatuses (aka 'cs')
 
item configmaps (aka 'cm')
 
item daemonsets (aka 'ds')
 
item deployments (aka 'deploy')
 
item deploymentconfigs (aka 'dc')
 
item endpoints (aka 'ep')
 
item events (aka 'ev')
 
item horizontalpodautoscalers (aka 'hpa')
 
item imagestreamimages (aka 'isimage')
 
item imagestreams (aka 'is')
 
item imagestreamtags (aka 'istag')
 
item ingresses (aka 'ing')
 
item groups
 
item jobs
 
item limitranges (aka 'limits')
 
item namespaces (aka 'ns')
 
item networkpolicies
 
item nodes (aka 'no')
 
item persistentvolumeclaims (aka 'pvc')
 
item persistentvolumes (aka 'pv')
 
item pods (aka 'po')
 
item podsecuritypolicies (aka 'psp')
 
item podtemplates
 
item policies
 
item projects
 
item replicasets (aka 'rs')
 
item replicationcontrollers (aka 'rc')
 
item resourcequotas (aka 'quota')
 
item rolebindings
 
item routes
 
item secrets
 
item serviceaccounts (aka 'sa')
 
item services (aka 'svc')
 
item statefulsets
 
item users
 
item storageclasses
 
item thirdpartyresources
 

OPTIONS

--include-extended-apis=true
If true, include definitions of new APIs via calls to the API server. [default true]
 
--recursive=false
Print the fields of fields (Currently only 1 level deep)

EXAMPLE

 
  # Get the documentation of the resource and its fields
  kubectl explain pods
  
  # Get the documentation of a specific field of a resource
  kubectl explain pods.spec.containers

NAME
kyua-atf-tester â Scriptable interface to interact with ATF test programs

SYNOPSIS
kyua-atf-tester [generic_flags] subcommand [subcommand_flags] [subcommand_args]

DESCRIPTION
kyua-atf-tester provides a scriptable interface to interact with test programs that implement the kyua-atf-interface(7) test interface. This manual page describes the specific
features that apply to this specific tester. For the generic manual page, which describes the command-line interface in detail, please see kyua-tester(1).

Results conversion
The result format outputted by kyua-atf-tester (described in kyua-tester-result(5)) is more restricted than what the ATF test case itself can output. The following list details
maps ATF test results to their generic results:

expected_death Maps to âexpected_failureâ if the death happens; âfailedâ otherwise.

expected_exit Maps to âexpected_failureâ if the test exits cleanly; âfailedâ otherwise.

expected_failure Maps to âexpected_failureâ if there is any failure during the execution; âfailedâ otherwise.

expected_signal Maps to âexpected_failureâ if the test terminates due to a signal; âfailedâ otherwise.

expected_timeout Maps to âexpected_failureâ if the test times out; âfailedâ otherwise.

failed Maps to âfailedâ.

passed Maps to âpassedâ.

skipped Maps to âskippedâ.

Any of the previous test results can also map to âbrokenâ in those cases where there is an unexpected problem in the communcation between the test and kyua-atf-tester. For examâ
ple, this will happen if result file created by the ATF test case does not comply to what kyua-atf-interface(7) describes.

Cleanup routines
Because kyua-atf-tester cannot keep any contextual information between the listing of test cases and their execution, the tester cannot easily (aka cheaply) know whether a test has
or has not a cleanup routine when it is run. For this reason, the âtestâ subcommand of kyua-atf-tester will assume, by default, that a test case does have a cleanup routine. This
postive assumption ensures that, if the tester is incorrectly executed, the test will be properly cleaned up.

However, as an optimization, the caller can pass â-v has_cleanup=falseâ to the âtestâ subcommand to tell kyua-atf-tester that the test being run does not have a cleanup routine.

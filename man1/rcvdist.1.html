NAME
rcvdist - asynchronously redistribute new mail

SYNOPSIS
/usr/libexec/nmh/rcvdist [-form formfile] [switches for postproc] address1 ... [-version] [-help]

DESCRIPTION
The rcvdist program will accept a message on its standard input and resend a copy of this message to all of the addresses listed on its command line.

When a message is redistributed with the rcvdist command, the format of the Resent-xxx header fields is controlled by the forms files ârcvdistcompsâ. If a file named ârcvdistâ
compsâ exists in the user’s nmh directory, it will be used instead of the default one. You may specify an alternate forms file with the switch -form formfile.

The ârcvdistcompsâ file uses the format string facility described in mh-format(5). In addition to the standard format escapes, rcvdist also recognizes the following additional
component escape:

Escape Returns Description
addresses string the addresses to distribute to

By default, rcvdist uses the program post to do the actual delivery of the message, although this can be changed by defining the postproc profile component.

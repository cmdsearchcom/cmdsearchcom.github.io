NAME
openshift infra deploy - Run the deployer

SYNOPSIS
openshift infra deploy [OPTIONS]

DESCRIPTION
Perform a deployment

This command launches a deployment as described by a deployment configuration. It accepts the name of a replication controller created by a deployment and runs that deployment to
completion. You can use the --until flag to run the deployment until you reach the specified condition.

Available conditions:

Â· "start": after old deployments are scaled to zero

Â· "pre": after the pre hook completes (even if no hook specified)

Â· "mid": after the mid hook completes (even if no hook specified)

Â· A percentage of the deployment, based on which strategy is in use

Â· "0%" Recreate after the previous deployment is scaled to zero

Â· "N%" Recreate after the acceptance check if this is not the first deployment

Â· "0%" Rolling before the rolling deployment is started, equivalent to "pre"

Â· "N%" Rolling the percentage of pods in the target deployment that are ready

Â· "100%" All after the deployment is at full scale, but before the post hook runs

Unrecognized conditions will be ignored and the deployment will run to completion. You can run this command multiple times when --until is specified - hooks will only be executed
once.
